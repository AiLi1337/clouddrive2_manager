# CloudDrive2 多实例一键部署与管理脚本 - 使用教程

## 环境要求

- 操作系统：Linux（推荐 Ubuntu 20.04+ / Debian 11+ / CentOS 8+）
- 权限：必须以 root 用户运行
- 依赖：Docker、Docker Compose（脚本会自动安装）
- 内核：需支持 FUSE（脚本会自动检测并尝试加载）

## 快速开始

```bash
# 1. 下载脚本
wget -O clouddrive2_manager.sh <脚本地址>

# 2. 赋予执行权限
chmod +x clouddrive2_manager.sh

# 3. 以 root 身份运行
sudo ./clouddrive2_manager.sh
```

## 启动流程

脚本启动后会自动执行以下初始化：

1. 检测是否以 `root` 运行
2. 检测并安装 `docker` 和 `docker-compose`（如未安装）
3. 检测 `/dev/fuse` 是否可用，不可用则尝试 `modprobe fuse`
4. 创建工作目录 `/opt/clouddrive2_manager` 及初始 `docker-compose.yml`

初始化完成后进入主菜单。

## 主菜单说明

```
  1) 添加新的 CD2 实例
  2) 删除已有的 CD2 实例
  3) 查看当前所有实例运行状态
  4) 一键更新所有实例
  5) 重启指定或全部实例
  6) 生成反向代理配置 (Caddy/Nginx)
  7) 卸载全部实例并清理数据
  0) 退出脚本
```

## 功能详解

### 1. 添加新的 CD2 实例

交互流程：

1. **输入实例名称**：仅允许英文字母、数字、下划线，如 `cd2_115`、`cd2_aliyun`
2. **输入宿主机端口**：脚本会自动检测未被占用的端口并推荐（默认从 19798 开始递增），直接回车采用推荐值
3. 脚本自动创建该实例的 `Config` 和 `CloudNAS` 目录
4. 自动追加服务配置到 `docker-compose.yml`
5. 启动容器并输出访问地址

示例：

```
Enter instance name (letters, numbers, underscores only): cd2_115
Enter host port [default: 19798]: ↵

[INFO] Instance 'cd2_115' is now running.
[INFO] Access URL: http://<your-ip>:19798
```

目录结构：

```
/opt/clouddrive2_manager/
├── docker-compose.yml
├── cd2_115/
│   ├── Config/
│   └── CloudNAS/
```

### 2. 删除已有的 CD2 实例

交互流程：

1. 列出当前所有实例，输入编号选择
2. 确认是否删除（y/N）
3. 选择是否同时删除本地持久化数据（Config 和 CloudNAS 目录）

- 选 **N**（默认）：仅删除容器和 compose 配置，数据保留在磁盘上
- 选 **Y**：彻底删除容器 + 配置 + 数据

### 3. 查看当前所有实例运行状态

以表格形式显示：

| 列 | 说明 |
|---|---|
| Instance | 实例名称 |
| Container ID | Docker 容器 ID |
| Status | 运行状态（Up / Exited） |
| Host Port | 映射到宿主机的端口 |
| Mem Usage | 内存占用 |

### 4. 一键更新所有实例

执行顺序：

1. `docker-compose pull` — 拉取最新镜像
2. `docker-compose up -d` — 使用新镜像重建并启动容器
3. `docker image prune -f` — 清理无用的旧镜像

> 更新过程中所有实例会短暂重启，请确保无正在进行的传输任务。

### 5. 重启实例

- 选择 **0**：重启全部实例
- 选择 **编号**：重启指定实例

### 6. 生成反向代理配置

交互流程：

1. **输入主域名**，如 `example.com`
2. **选择反代类型**：`1) Caddy` 或 `2) Nginx`
3. 脚本遍历所有运行中的实例，为每个实例分配子域名并生成配置文件

生成的子域名规则：`<实例名>.<主域名>`

例如域名为 `example.com`，实例为 `cd2_115`，则子域名为 `cd2_115.example.com`

**Caddy 配置** 生成到：`/opt/clouddrive2_manager/Caddyfile`

```
cd2_115.example.com {
    reverse_proxy localhost:19798
}
```

**Nginx 配置** 生成到：`/opt/clouddrive2_manager/cd2_proxy.conf`

部署方式：

```bash
# Caddy
cp /opt/clouddrive2_manager/Caddyfile /etc/caddy/
systemctl reload caddy

# Nginx
cp /opt/clouddrive2_manager/cd2_proxy.conf /etc/nginx/conf.d/
nginx -t && nginx -s reload
```

### 7. 卸载全部实例并清理数据

> ⚠️ 此操作不可逆！

交互流程：

1. 提示警告
2. 需要手动输入 **`YES`** 确认（其他任何输入均取消）
3. 执行 `docker-compose down` 停止并删除所有容器
4. 删除整个工作目录 `/opt/clouddrive2_manager`

## 常见问题

### Q: /dev/fuse 不可用怎么办？

某些 VPS（如 OpenVZ 架构）不支持 FUSE，CD2 的挂载功能将无法使用。建议更换为 KVM 架构的 VPS。

### Q: 端口被占用无法启动？

脚本添加实例时会自动检测端口占用并推荐可用端口。如果已被其他服务占用，请手动指定其他端口。

### Q: 如何手动修改某实例的配置？

编辑 `/opt/clouddrive2_manager/docker-compose.yml`，修改后执行：

```bash
cd /opt/clouddrive2_manager
docker-compose up -d <实例名>
```

### Q: 数据存储在哪里？

所有实例数据位于 `/opt/clouddrive2_manager/<实例名>/` 下：

- `Config/` — CD2 配置文件
- `CloudNAS/` — CD2 挂载点数据

### Q: 支持哪些 Linux 发行版？

支持主流发行版（Ubuntu / Debian / CentOS 等），只要是能运行 Docker 的 Linux 系统均可。脚本使用 Docker 官方安装脚本自动安装 Docker，不支持的发行版需手动安装。
